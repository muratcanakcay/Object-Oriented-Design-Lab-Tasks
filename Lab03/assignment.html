The data

The document has to store information about tortoises vs snails racing competition. The competition is described by:

  • racing circuits,
  • competitors,
  • event’s descriptions (joining circuits with particular competitors and dates).

You are provided with a sample XML file conforming to the requested format.

The document root element is named formulaT. The element has the following children: participants, track, event. There is only one participants element and it is always the first one. Track and event element come after participants in no particular order. There can be two or more tracks and events and they can be interleaved with each other.

The structure of child elements is as follows:

  • participants – describes all participants of the competition – turtles and snails. It contains zero or more tortoise and snail elements. All tortoise elements must precede snail elements.

    • tortoise has a single compulsory attribute startNumber that contains a participant’s start number in format a capital T letter and two digits. The value must be unique across all participants (both snails and tortoises) in the document. It has three child elements in exactly that order:

      • nick – a nick of the participant – a string
      • weight – a decimal number

      • color – one of the three possible color values:

        • green
        • dark green
        • light green

    • snail has a single compulsory attribute startNumber that contains a participant’s start number in format a capital T letter and two digits. The value must be unique across all participants (both snails and tortoises) in the document. It has three child elements in exactly that order:

      • nick – a nick of the participant – a string
      • weight – a decimal number
      • either slow or fast element. (exactly one of those). This element must be empty.

  • track contains the three following elements in exactly that order

    • id – a positive integer that uniquely identifies a track in the document (it must be unique across the all tracks).
    • name – a string
    • length – a decimal number

  • event

    • name – a string that starts with a capital letter and contains at least two capital letters in total.
    • date – a date in format YYYY-MM-DD
    • track – element with a single mandatory attribute id, a number. This must be a id of a track defined in the document.
    • two or more participant elements. Each participant elements has mandatory id attribute. This attribute must point to a existing participant’s start number. Moreover within a single event participants must be unique.
